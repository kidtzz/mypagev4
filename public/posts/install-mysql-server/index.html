Install MYSQL Server

December 9, 2024 · 2 min · 291 words · kidtz | Edit

prereq#

  1. Get repository

    wget https://repo.mysql.com//mysql57-community-release-el7-11.noarch.rpm
sudo rpm -ivh mysql57-community-release-el7-11.noarch.rpm

  2. Get key mysql

    rpm --import https://repo.mysql.com/RPM-GPG-KEY-mysql-2022
rpm -qa gpg-pubkey*
ls -l /etc/pki/rpm-gpg/RPM-GPG-KEY-mysql

Install Mysql#

  1. Install mysql-server

    sudo yum install mysql-server

  2. Because MySQL is automatically installed you can use OS command for service management, for example to check if it’s already started

    sudo systemctl status mysqld

  3. Start MySQL if not started

    sudo systemctl start mysqld
sudo systemctl enable mysqld
sudo systemctl status mysqld

Change root password#

  1. Check the content of my.cnf, that is in default folder for linux OS and note the following info (lines that start with “#”are just comments)
  • Where is the database and the error log (mysqld.log) stored?
  • Check if there are error for the instance looking in the error log file

  1. Starting from MySQL 5.7 the default installation of MySQL Server generates a one-time password. You find it in error log notes above

    sudo grep 'temporary' /var/log/mysqld.log

  2. Login to MySQL using password retrieved in previous step

    mysql -uroot -p -h localhost

  3. Try to run a command and write down the error message

    mysql>status

  4. Change root password

    mysql>ALTER USER 'root'@'localhost' IDENTIFIED BY 'Welcome1!';

  5. Retry command above, now it works

    mysql>status;

  6. Which databases are installed by default?

    mysql>show databases;

  7. To see which version of MySQL you are using submit the command

    mysql>show variables like "%version%";

  8. Check default users in standard installation

    mysql>select user, host from mysql.user where user='root';

  9. Logout as ‘root’ and connect as admin

    mysql>exit
